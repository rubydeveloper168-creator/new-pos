Add Category Migration to Data Migration Tool
This change adds category migration functionality to the existing data migration page, migrating categories before products so products can have their correct category_id mapped.

Proposed Changes
Migration Controller
[MODIFY] 
MigrationUpdateController.php
Add migrateCategories() method - New private method to:

Fetch categories from old POS sma_categories table
Map old code → new short_code
Map old name → new name
Map old parent_id → new parent_id (maintaining hierarchy)
Set category_type = 'product'
Set business_id from session
Track old_id → new_id mapping for product migration
Modify 
migrateProducts()
 method - Update to:

Accept category mapping parameter
Lookup old product's category_id in old sma_products table
Map to new category ID using the category mapping
Set category_id on new product record
Modify 
runMigration()
 method - Update to:

Call migrateCategories() BEFORE 
migrateProducts()
Pass category mapping to 
migrateProducts()
Include category count in SSE progress messages
Modify 
cleanMigratedData()
 method - Update to:

Add step to delete migrated categories (optional, only if user wants to clean categories)
Migration View
[MODIFY] 
index.blade.php
Update info box - Add "Categories" to the migration includes list
Add Categories statistics box - New info-box to show category count
Update JavaScript extractCounts() function - Parse category count from log messages
Field Mapping
Old POS (sma_categories)	New POS (categories)
id	Track for mapping
code	short_code
name	name
parent_id	parent_id (remapped to new IDs)
image	Not used (no image field in new table)
slug	Not used
description	Not used (can use category_type note if needed)
-	business_id (from session)
-	category_type = 'product'
-	created_by = 1
Verification Plan
Manual Verification
Since this is a database migration tool with SSE streaming, manual testing is required:

Navigate to the migration page: http://localhost:8888/migrate-update-data (or your configured host)
Click "Clean Migrated Data" first to start fresh
Click "Start Migration"
Verify in the log output:
"Fetching categories from old database..." message appears
"Found X categories to migrate" shows correct count
"Migrated X categories" confirmation
Categories are migrated BEFORE products
Check the Categories statistics box shows the migrated count
Verify in database:
Query new POS: SELECT * FROM categories WHERE category_type = 'product'
Compare with old POS: SELECT * FROM sma_categories
Check parent_id relationships are preserved
Verify products have categories:
Query: SELECT id, name, category_id FROM products WHERE category_id IS NOT NULL LIMIT 10
Cross-reference category_id with migrated categories
